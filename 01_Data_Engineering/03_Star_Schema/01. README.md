# Star Schema

## Overview

This project follows the Kimball Dimensional Modeling approach by implementing a Star Schema for payment transaction analytics.

The design separates descriptive attributes into dimension tables while storing transactional measures inside a centralized fact table.

This architecture improves:

- Query performance
- Scalability
- Business reporting
- Analytical workloads
- Power BI integration

---

## Fact Table

- Fact_PaymentTransactions

---

## Dimension Tables

- Dim_Time
- Dim_TransactionType
- Dim_Fraud
- Dim_SourceAccount
- Dim_DestinationAccount

---

## Relationships

Every dimension has a one-to-many relationship with the Fact table through surrogate keys.

This schema supports enterprise-scale analytical queries on more than 6 million payment transactions.
